Хотите ли вы писать приложения, используя свой любимый веб-фреймворк, управлять контентом через панель 
администратора, получая на выходе полностью валидные страницы c поддержкой последних рекомендаций SEO, PWA и доступности интерфейса. 
Добро пожаловать под кат!

## Что же такое JAMstack?

> JAM = JavaScript + APIs + Markup

Несмотря на название, это не набор инструментов, а общий подход к проектированию приложений, который состоит из трёх компонентов:

- JavaScript –  мультипарадигменный язык программирования, нужный нам для разработки браузерного интерактива. 
Для совсем простых сайтов можно обойтись и без него
- APIs – Источники данных из которых будут генерироваться страницы. В качестве таких источников может выступать что угодно: 
базы данных, системы управления контентом или обычные текстовые файлы
- Markup – Буквально, это разметка сайта. На примитивном уровне это связанные между собой html файлы. 
Но зачастую подразумевается использование генераторов статичных сайтов, которые создают страницы из 
полученных данных по заранее определённому шаблону

## Архитектура

Справедливо заметить, что такой подход похож на то, как разрабатывали сайты лет 10 назад, когда сервер по запросу 
от клиента вытягивал данные из базы, рендерил на их основе статику и оправлял её на клиент. 

Разница состоит в том, мы генерируем сразу весь контент, который можем разместить на простом файловом сервере. 
Когда мы обновляем слой данных, нам нужно заново запустить генератор статики, чтобы обновить слой представления. К счастью, 
современные хостинги предоставляют удобные инструменты для автоматического повторного развертывания проекта в зависимости 
от внешних условий, например: коммита в репозиторий с исходниками или добавление записи в облачную CMS. 
Мы можем полностью отказаться от разработки собственной серверной логики, используя инфраструктуру облачных сервисов.

## Преимущества использования JAMstack

- Отказоустойчивость
- Безопасность
- Легкая масштабируемость
- Стабильное быстрое время отклика
- Высокий аптайм
- Сокращение времени разработки приложения
- Снижение расходов на долгосрочную поддержку приложения

## Ограничения использования JAMstack

- Процесс обновления данных на сайте занимает много времени (до 10 минут, в зависимости от объема приложения и хостинга), 
из-за того, что модификация контента вызывает повторное развертывание всего проекта
- Данные и исходный код хранятся в облаке, что в некоторых ситуациях может быть неприемлемо

Если вам нужна динамическая обработка запросов пользователя и обновление данных на лету,
то лучше использовать традиционную архитектуру приложения.

Весь функционал облачных сервисов можно перенести на собственные сервера. Например, развернуть базу данных 
и систему управления контентом на подконтрольной машине вместо использования готового сервиса. 
В этом случае ответственность за конфиденциальность или доступность данных лежит только на вас самих.

Подводя итог, JAMstack отлично подходит для разработки личных страниц, портфолио, блогов, документации и 
любых других статичных сайтов, где скорость обновления контента отходит на второй план.
